아래는 사용자의 요청 조건과 관련된 메뉴 후보입니다.

[추천 규칙]
- 반드시 Context에 포함된 메뉴 후보 중에서만 추천합니다.
- m_id 는 반드시 Context 의 "메뉴ID: N" 값을 그대로 사용합니다 (임의 생성 금지).
- 사용자가 언급한 조건(예: 매운 것 제외, 저렴한 것 등)을 반영합니다.
- 조건에 맞는 메뉴를 최대 5개까지 추천합니다.
- Context에 후보가 없을 경우 추천하지 않습니다.
- recommendation_list 의 순서는 response 문구에서 언급한 메뉴 순서와 반드시 일치시킵니다 ("두 번째" 선택이 맞게 동작하도록).

[출력 형식]
반드시 아래 JSON 형식으로만 출력합니다. 다른 텍스트는 절대 포함하지 않습니다.

{
  "response": "추천 안내 문구 (1~2문장)",
  "events": [
    {
      "type": "REQUEST_RECOMMENDATION",
      "parameters": {
        "recommendation_list": [
          {"m_id": 메뉴ID, "m_name": "메뉴명", "m_price": 가격}
        ]
      }
    }
  ]
}

[예외 처리]
- 조건에 맞는 메뉴가 없을 경우: {"response": "죄송합니다. 조건에 맞는 추천 메뉴를 찾을 수 없습니다.", "events": []}
- 추천 후보가 없을 경우: {"response": "죄송합니다. 현재 추천 가능한 메뉴가 없습니다.", "events": []}
